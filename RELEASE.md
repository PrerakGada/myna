# RELEASE.md — how to ship Myna

This is the operator manual for the release pipeline. After the one-time
setup below, every release is a two-command operation:

```bash
git tag v0.1.0 && git push --tags
```

…which triggers `.github/workflows/release.yml` and produces a signed,
notarized DMG, attaches it to a GitHub Release, refreshes the Sparkle
appcast, and bumps the Homebrew cask.

---

## 1. One-time setup (do this once, before the first release)

### 1.1 Apple Developer ID

You need a paid Apple Developer account ($99/yr).

1. Visit <https://developer.apple.com/account/resources/certificates/list>.
2. Create a **Developer ID Application** certificate.
3. Download it and double-click to import into your login keychain.
4. In Keychain Access, right-click the cert → **Export…** → save as
   `Myna-DeveloperID.p12` with a strong password (you'll need this password
   in step 1.4 below).

### 1.2 App-specific password

Notarytool needs an app-specific password (not your regular Apple ID password).

1. Sign in at <https://appleid.apple.com> → **Sign-In and Security** →
   **App-Specific Passwords** → **+**.
2. Name it `Myna notarization`. Save the generated string.

### 1.3 Team ID

Find your 10-character Team ID at
<https://developer.apple.com/account#MembershipDetailsCard>.

### 1.4 Sparkle EdDSA keys

This repo ships with keys already generated by Lane B (overnight session).
The public key is baked into `apps/macos/project.yml` (`SUPublicEDKey`). The
**private** key lives at `dist/sparkle_private_key.NEVER_COMMIT.txt` (which
is git-ignored).

Action required:

1. **Stash the private key in 1Password** as a secure note named
   `Myna Sparkle EdDSA private key`. Copy the entire one-line base64 string.
2. **Delete the file from disk** once it's in 1Password:
   ```bash
   rm dist/sparkle_private_key.NEVER_COMMIT.txt
   ```
3. **Never** rotate this key after the first public release. If you do,
   every installed copy of Myna will refuse all future updates (Sparkle
   verifies the signature against the public key baked into the .app at
   build time, and old installs can't be patched). If the key is leaked,
   you can ship a new minor version with a new public key — but every user
   then has to download the new build manually once.

If you ever need to regenerate the keys (only for a brand new app), the
canonical tool is Sparkle's `generate_keys`:

```bash
# After `git clone https://github.com/sparkle-project/Sparkle && cd Sparkle`:
./bin/generate_keys
# Or via brew:
brew install --cask sparkle && /opt/homebrew/Caskroom/sparkle/*/bin/generate_keys
```

Lane B generated this repo's keys using OpenSSL (Ed25519 with the 32-byte
raw key encoded as base64 — identical format to Sparkle's tool).

### 1.5 Homebrew tap repository

`release.yml`'s `tap-bump` job pushes to a separate `homebrew-tap` repo.
You create that repo once and point a deploy key at it.

```bash
gh repo create PrerakGada/homebrew-tap --public --description "Myna's Homebrew tap"
git clone git@github.com:PrerakGada/homebrew-tap.git ~/Developer/homebrew-tap
cp -R tap/Casks tap/Formula ~/Developer/homebrew-tap/
cd ~/Developer/homebrew-tap
git add -A && git commit -m 'initial' && git push
```

Generate a deploy key with write access:

```bash
ssh-keygen -t ed25519 -f ~/.ssh/myna-tap-deploy -N "" -C "myna-tap-deploy"
gh repo deploy-key add ~/.ssh/myna-tap-deploy.pub \
  --repo PrerakGada/homebrew-tap \
  --allow-write \
  --title "release.yml"
# Copy the PRIVATE key (the file WITHOUT .pub) — you'll paste it into
# GitHub secrets as TAP_DEPLOY_KEY below.
cat ~/.ssh/myna-tap-deploy
```

### 1.6 GitHub Actions secrets

Set these under **Repo → Settings → Secrets and variables → Actions**:

| Secret name | Value |
|---|---|
| `APPLE_DEVELOPER_ID_P12` | `base64 -i ~/Downloads/Myna-DeveloperID.p12 \| pbcopy` — paste that base64 string |
| `APPLE_DEVELOPER_ID_P12_PASSWORD` | the password you set in step 1.1 |
| `APPLE_DEVELOPER_ID_NAME` | `Developer ID Application: MIND WEALTH (RC63N3VU27)` — find via `security find-identity -v -p codesigning` |
| `APPLE_ID` | your Apple ID email (step 1.2) |
| `APPLE_ID_APP_PASSWORD` | the app-specific password from step 1.2 |
| `APPLE_TEAM_ID` | the 10-char team ID from step 1.3 |
| `KEYCHAIN_PASSWORD` | any random string (e.g. `openssl rand -base64 24`) — used to create a temp keychain during CI |
| `SPARKLE_EDDSA_PRIVATE_KEY` | the base64 string from step 1.4 (the contents of `sparkle_private_key.NEVER_COMMIT.txt`) |
| `TAP_DEPLOY_KEY` | the private SSH key from step 1.5 |

The `release.yml` `preflight` job fails fast with the exact missing secret
names if any of these are absent, so it's safe to try a tag push to see
what's missing.

### 1.7 Update the `CHANGEME` placeholders

Search and replace `CHANGEME` for your GitHub username/owner across:
- `tap/Casks/myna.rb`
- `tap/Formula/myna-daemon.rb`
- `apps/macos/project.yml` (`SUFeedURL`)
- `dist/appcast.sh` (`DOWNLOAD_BASE_URL` default)

The workflows use `${{ github.repository }}` automatically, so they don't
need editing.

## 2. Per-release procedure

Once all of section 1 is done, releasing is:

```bash
# 1. Bump the version in project.yml (MARKETING_VERSION)
# 2. Commit it
git commit -am "release: v0.1.0"
# 3. Tag and push
git tag v0.1.0
git push origin main --tags
```

The workflow takes ~10–20 minutes end-to-end (the notarize step is the
slowest — Apple's queue varies wildly).

While it's running, watch:

```bash
gh run watch
# or
gh run list --workflow release.yml --limit 1
```

When green, verify:

```bash
gh release view v0.1.0
brew tap PrerakGada/tap
brew install --cask myna   # should install the brand-new DMG
```

### 2.1 First release vs subsequent releases

The very first release has no existing appcast. `release.yml` handles this
gracefully — the `appcast` job's `gh release download appcast …` step is
allowed to 404, and `dist/appcast.sh` creates a fresh `appcast.xml` from
scratch.

After that first release, the dedicated `appcast` release exists and every
subsequent run downloads, appends, re-uploads.

## 3. Rollback

A bad release happens. Here's the recovery procedure:

```bash
# Delete the GitHub release + its tag.
gh release delete v0.1.0 --yes
git push --delete origin v0.1.0
git tag -d v0.1.0

# Revert the tap commit so users on `brew upgrade` don't pull the bad build.
cd ~/Developer/homebrew-tap
git log --oneline       # find the bad commit
git revert <bad-sha>
git push

# Refresh the appcast so existing Myna installs don't see the bad version.
gh workflow run appcast.yml --field count=20
# After it succeeds, manually inspect the appcast.xml asset on the `appcast`
# release to confirm the bad version is gone.
```

Sparkle clients that already pulled the bad update will need a manual
re-install (download the prior DMG from the GitHub releases page). There's
no way to "un-ship" an auto-update once Sparkle has installed it on a
user's machine — only ship a fixed version on top.

## 4. Manual notarization fallback

If `release.yml` is broken (Apple changes the API, GitHub Actions outage,
etc.) you can run the full pipeline locally:

```bash
# 1. Build the .app
VERSION=0.1.0 bash dist/build.sh

# 2. Set Developer ID env, then sign
export DEVELOPER_ID_APPLICATION="Developer ID Application: MIND WEALTH (RC63N3VU27)"
bash dist/sign.sh

# 3. Set Apple ID env, then notarize the .app
export APPLE_ID="prerak@engaze.in"
export APPLE_TEAM_ID="RC63N3VU27"
export APPLE_ID_APP_PASSWORD="abcd-efgh-ijkl-mnop"
bash dist/notarize.sh

# 4. Wrap in DMG
brew install create-dmg   # one-time
VERSION=0.1.0 bash dist/dmg.sh

# 5. Sign the DMG, then notarize that too
codesign --force --sign "$DEVELOPER_ID_APPLICATION" --timestamp \
  dist/out/Myna-0.1.0.dmg
TARGET=dist/out/Myna-0.1.0.dmg TARGET_SET=1 bash dist/notarize.sh

# 6. Sign the Sparkle appcast entry
export SPARKLE_EDDSA_PRIVATE_KEY="$(op read 'op://Private/Myna Sparkle EdDSA/password')"
VERSION=0.1.0 \
  DOWNLOAD_BASE_URL="https://github.com/PrerakGada/myna/releases/download/v0.1.0" \
  bash dist/appcast.sh

# 7. Publish the GH release manually
gh release create v0.1.0 dist/out/Myna-0.1.0.dmg \
  --title "Myna 0.1.0" \
  --notes "Manual release — see dist/out/appcast.xml for Sparkle metadata"

# 8. Update the appcast release
gh release upload appcast dist/out/appcast.xml --clobber

# 9. Bump the cask manually
cd ~/Developer/homebrew-myna
SHA=$(shasum -a 256 ../myna/dist/out/Myna-0.1.0.dmg | awk '{print $1}')
sed -i.bak -E 's/^( *version )"[^"]+"/\1"0.1.0"/' Casks/myna.rb
sed -i.bak -E "s/^( *sha256 )(:no_check|\"[^\"]+\")/\1\"$SHA\"/" Casks/myna.rb
rm Casks/myna.rb.bak
git commit -am 'myna 0.1.0' && git push
```

Every script also supports `--dry-run` if you want to walk through it without
hitting Apple infrastructure first.

## 5. Useful commands

```bash
# Find your Developer ID identity string for APPLE_DEVELOPER_ID_NAME
security find-identity -v -p codesigning

# Smoke-test all scripts locally
bash dist/tests/test_scripts.sh

# Validate workflows
python3 -c "import yaml, glob; [yaml.safe_load(open(f)) for f in glob.glob('.github/workflows/*.yml')]"

# Audit the tap formulas
brew style tap/Casks/myna.rb
brew style tap/Formula/myna-daemon.rb

# Verify a notarized .app
spctl --assess --type execute --verbose=2 /Applications/Myna.app
codesign --verify --deep --strict --verbose=2 /Applications/Myna.app
stapler validate /Applications/Myna.app

# Sparkle verify a DMG against the public key (sanity)
# (requires Sparkle's `sign_update -v` tool installed)
sign_update -v -f <(echo "$SPARKLE_EDDSA_PRIVATE_KEY") /path/to/Myna-0.1.0.dmg
```
