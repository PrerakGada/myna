# What's new in Myna 0.2

## Highlights

- **Faster start.** Priority-first chunking with a rolling prefetch —
  Myna now begins speaking around **240 ms** after you press the
  shortcut, instead of sitting silent for a beat first.
- **Beautiful new popover.** A custom SwiftUI menu-bar surface
  replaces the old NSMenu — near-black background, soft cards,
  monospaced time codes that stop jittering, a Recent submenu of the
  last five things Myna read.
- **A thoughtful bird.** The menu-bar icon shifts through five states
  (idle, speaking, thinking, paused, error) — quietly, at ~0.2% CPU.
- **Karaoke ribbon (opt-in).** A companion sidecar shows the current
  spoken line at the bottom of your screen, word by word. Toggle from
  the menu. Lives in its own signed bundle so the main app stays calm.
- **CC toast cards.** When a Claude Code cloud hook fires, a small
  playable card slides in from the top-right of your screen —
  project-tinted dot, one-tap Play, auto-tucks into the menu bar
  after 8 s if you ignore it. Stacks up to three.
- **Voice previews.** Try a voice from Settings → Voice before
  committing to it. Hovering ducks any in-flight audio.
- **Quieter bird.** Fixed a runaway CPU bug where the animated icon
  burned ~99.5% of a core. The icon is now a static SF Symbol per
  state — no fans, no battery hit.
- **Floating pill (beta).** A Wispr Flow-style transport hovers at
  the bottom of the active display while Myna is speaking — collapsed
  by default, hover to expand into a full mini-player. Toggle from
  Settings → Advanced. Hides itself the moment audio ends.
- **Trackpad gestures (beta).** Opt-in 4-finger swipe to seek
  ±30 seconds, force-touch click to pause/resume. Settings → Gestures.
  Default OFF so it never surprises you.
- **Voice wardrobe (beta).** Pin a different voice per app — Bella
  for articles, Anna for code, Joe for chat. Set rules in
  Settings → Wardrobe; Myna applies them automatically when you
  trigger speak from each app.
- **Quiet language detector.** Myna now notices when the text you
  asked it to read isn't English (langid, on-device) and surfaces a
  small "Detected: ES — adjust voice in Settings" hint in the popover.
  Never auto-switches voices on you.

## Behind the scenes

- AVAudioEngine ducking for in-flight voice previews
- Adaptive 250 ms / 1 s status polling so the popover stays live
  without draining the battery
- Inside-out signing for the karaoke sidecar (Gatekeeper-happy
  nested bundle)
- Engine-down state now surfaces in the bird icon instead of failing
  silent

## Coming next

- **Per-word timing.** The karaoke ribbon will eventually highlight
  one word at a time instead of one line.
- **Voice preview ▶ in tiles.** Quick auditions without leaving the
  Voice tab.
- **Wardrobe quick-add.** "Always use this voice for Xcode" from the
  current popover, no Settings round-trip.
